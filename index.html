Sección 1

¿Qué es HTML y CSS?

🏗️ HTML — La estructura

HTML (HyperText Markup Language) es el esqueleto de toda página web. Define el contenido: títulos, párrafos, imágenes, botones, etc. Sin HTML, no hay página web.

🎨 CSS — El estilo

CSS (Cascading Style Sheets) es el maquillaje de la página web. Define colores, tamaños, fuentes, espacios... Convierte una página aburrida en algo bonito.

💡 Analogía del día Imagina una casa: HTML es la estructura (paredes, puertas, ventanas) y CSS es la decoración (color de las paredes, muebles, iluminación). Juntos hacen que la casa sea habitable y bonita.

¿Cómo funciona una página web?

1

Tú escribes código HTML y CSS

En un editor de texto creas los archivos .html y .css

2

El navegador lee tu código

Chrome, Firefox o Edge interpretan el código y lo convierten en la página visual

3

¡El usuario ve tu página!

Cualquier persona puede abrir tu archivo en su navegador y ver el resultado

Sección 2

Mi primer archivo HTML

Todo archivo HTML empieza con la misma estructura base. Es como el "molde" que siempre debes usar.

📄 mi-primera-pagina.html 
<!-- Esta línea le dice al navegador que es HTML5 -->
<!DOCTYPE html>

<!-- El idioma de la página (es = español) -->
<html lang="es">

  <head>
    <!-- El tipo de caracteres (permite tildes y ñ) -->
    <meta charset="UTF-8">
    <!-- Para que se vea bien en celulares -->
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <!-- El título que aparece en la pestaña del navegador -->
    <title>Mi primera página</title>
  </head>

  <body>
    <!-- Todo lo visible va aquí -->
    <h1>¡Hola, mundo!</h1>
    <p>Esta es mi primera página web.</p>
  </body>

</html>

🧠 <head> — La cabeza

Contiene información para el navegador: título de la página, codificación de caracteres, enlace al CSS. El usuario NO ve este contenido directamente.

👁️ <body> — El cuerpo

Todo lo que el usuario ve en pantalla va aquí: textos, imágenes, botones, listas. Este es el área de trabajo principal.

⚠️ Regla importante Las etiquetas HTML son como paréntesis: siempre deben abrirse y cerrarse. Si abres <p>, debes cerrarlo con </p>. La barra inclinada "/" indica el cierre.
Ejercicio 1

¡Tu primera página!

  1. Abre VS Code y crea un nuevo archivo llamado index.html
  2. Copia la estructura base de arriba
  3. Cambia el título a tu nombre
  4. En el <body>, escribe: <h1>Hola, soy [tu nombre]</h1>
  5. Guarda el archivo (Ctrl+S) y ábrelo con el navegador

🎯 Objetivo: Ver tu nombre como título en el navegador.

Sección 5

Enlaces e imágenes

🔗 La etiqueta <a> — Hipervínculos

La etiqueta <a> (anchor) crea enlaces que nos llevan a otras páginas.

HTML — Tipos de enlaces
<!-- Enlace a otra página web -->
<a href="https://www.google.com">Ir a Google</a>

<!-- Enlace que abre en pestaña nueva -->
<a href="https://www.google.com" target="_blank">Google (nueva pestaña)</a>

<!-- Enlace a otra página de TU sitio -->
<a href="contacto.html">Ir a Contacto</a>

<!-- Enlace para enviar un correo -->
<a href="mailto:hola@ejemplo.com">Enviar correo</a>

🖼️ La etiqueta <img> — Imágenes

La etiqueta <img> muestra imágenes. No tiene etiqueta de cierre.

HTML — Imágenes
<!-- Imagen desde Internet -->
<img src="https://picsum.photos/300/200" 
     alt="Descripción de la imagen"
     width="300">

<!-- Imagen desde tu carpeta (mismo directorio) -->
<img src="mi-foto.jpg" 
     alt="Mi foto"
     width="400">
🚨 El atributo alt es obligatorio El alt es el texto alternativo que aparece si la imagen no carga, y lo usan lectores de pantalla para personas con discapacidad visual. Siempre ponlo.
AtributoSignificadoEjemplo

altTexto descriptivo alternativoalt="Un gato durmiendo"
widthAncho en píxeleswidth="300"
heightAlto en píxelesheight="200"

Sección 6

Tablas

Las tablas organizan información en filas y columnas. Son útiles para horarios, comparaciones, datos, etc.

HTML — Estructura de tabla
<table>
  <!-- thead = cabecera de la tabla -->
  <thead>
    <tr>  <!-- tr = fila (table row) -->
      <th>Nombre</th>   <!-- th = celda encabezado -->
      <th>Materia</th>
      <th>Nota</th>
    </tr>
  </thead>
  
  <!-- tbody = cuerpo de la tabla -->
  <tbody>
    <tr>
      <td>Ana López</td>  <!-- td = celda normal -->
      <td>Matemáticas</td>
      <td>90</td>
    </tr>
    <tr>
      <td>Pedro García</td>
      <td>Ciencias</td>
      <td>85</td>
    </tr>
  </tbody>
</table>
Resultado:
Nombre Materia Nota
Ana López Matemáticas 90
Pedro García Ciencias 85

Sección 8

HTML Semántico

HTML5 introdujo etiquetas que describen el significado del contenido, no solo su apariencia. Esto ayuda a los buscadores como Google a entender tu página.

HTML — Estructura semántica de una página
<header>
  <!-- Logo, nombre del sitio, menú principal -->
  <h1>Mi Sitio Web</h1>
  <nav>
    <a href="index.html">Inicio</a>
    <a href="sobre-mi.html">Sobre mí</a>
  </nav>
</header>

<main>
  <article>
    <!-- Un artículo o entrada del blog -->
    <h2>Mi primer artículo</h2>
    <p>Contenido del artículo...</p>
  </article>
  
  <aside>
    <!-- Contenido secundario: publicidad, links relacionados -->
    <p>Artículos relacionados...</p>
  </aside>
</main>

<footer>
  <!-- Pie de página: derechos, redes sociales, contacto -->
  <p>© 2025 Mi Nombre. Todos los derechos reservados.</p>
</footer>

🔝 <header>

Encabezado de la página o sección. Suele contener el logo y la navegación principal.

🧭 <nav>

Menú de navegación. Contiene los enlaces principales del sitio.

📖 <main>

Contenido principal y único de la página. Solo debe haber uno por página.

📰 <article>

Contenido independiente: un artículo, entrada de blog, publicación.

📌 <aside>

Contenido secundario o relacionado. Barras laterales, datos curiosos.

🔻 <footer>

Pie de página. Derechos, redes sociales, información legal.

Sección 9

¿Cómo funciona CSS?

CSS funciona con una sintaxis simple: seleccionas algo y le das propiedades.

Anatomía de una regla CSS
p {                    /* selector: ¿a qué aplico el estilo? */
  color: blue;          /* propiedad: valor; */
  font-size: 18px;      /* otra propiedad */
  background-color: yellow;
}

¿Cómo conecto CSS con HTML?

1. Archivo externo ⭐

Lo más recomendado. Crea un archivo styles.css y enlázalo en el <head>:

<link rel="stylesheet"
      href="styles.css">

2. En el <head>

Dentro de etiquetas <style> en el HTML:

<style>
  p { color: red; }
</style>

3. En línea

Directamente en la etiqueta (menos recomendado):

<p style="color:red">
  Texto
</p>

Sección 10

Selectores CSS

Los selectores determinan a qué elementos se aplican los estilos. Hay varios tipos:

styles.css — Tipos de selectores
/* 1. Selector de ETIQUETA — aplica a TODOS los p */
p {
  color: #333;
}

/* 2. Selector de CLASE — aplica a class="destacado" */
.destacado {
  background-color: yellow;
  font-weight: bold;
}

/* 3. Selector de ID — aplica a id="titulo-principal" */
#titulo-principal {
  font-size: 36px;
  text-align: center;
}

/* 4. Selector DESCENDIENTE — p dentro de un div */
div p {
  margin: 10px;
}

/* 5. Múltiples elementos a la vez */
h1, h2, h3 {
  font-family: Arial, sans-serif;
}

🏷️ Clases (.clase)

Son el tipo más usado. Puedes aplicar la misma clase a múltiples elementos.

/* CSS */
.boton-azul {
  background: blue;
  color: white;
}

<!-- HTML -->
<button class="boton-azul">Click</button>
<p class="boton-azul">¡Yo también!</p>

🆔 IDs (#id)

Únicos por página. Solo un elemento puede tener cada ID. Úsalos con moderación.

/* CSS */
#menu {
  background: darkblue;
  padding: 20px;
}

<!-- HTML -->
<nav id="menu">
  ...menú aquí...
</nav>

Sección 11

Colores y fondos

CSS admite varios formatos para escribir colores:

CSS — Formas de escribir colores
p {
  /* Por nombre (inglés) */
  color: red;
  color: blue;
  color: hotpink;

  /* Hexadecimal (el más usado por diseñadores) */
  color: #ff0000;   /* rojo */
  color: #3498db;   /* azul bonito */
  color: #2ecc71;   /* verde */

  /* RGB (rojo, verde, azul — de 0 a 255) */
  color: rgb(255, 0, 0);     /* rojo */
  color: rgb(52, 152, 219);  /* azul */

  /* RGBA — como RGB pero con transparencia (0=invisible, 1=sólido) */
  color: rgba(255, 0, 0, 0.5);  /* rojo semitransparente */
}

Sección 13

El Box Model — La caja de CSS

En CSS, todo elemento es una caja rectangular. Cada caja tiene 4 capas:

MARGIN (espacio exterior)
BORDER (borde)
PADDING (relleno interior)
CONTENT (el contenido)
CSS — Box Model completo
.mi-caja {
  /* Dimensiones del contenido */
  width: 300px;
  height: 200px;

  /* PADDING: espacio ADENTRO del borde (4 lados) */
  padding: 20px;              /* todos iguales */
  padding: 10px 20px;         /* arriba/abajo izq/der */
  padding-top: 15px;          /* solo arriba */

  /* BORDER: el borde visible */
  border: 2px solid black;    /* grosor tipo color */
  border-radius: 10px;        /* esquinas redondeadas */

  /* MARGIN: espacio AFUERA del borde */
  margin: 30px;
  margin: auto;               /* centra horizontalmente */
  margin-bottom: 20px;

  /* IMPORTANTE: incluye padding en el tamaño total */
  box-sizing: border-box;
}

📏 Padding vs Margin

Padding: espacio entre el contenido y el borde. Es parte del elemento (tiene el color de fondo). Margin: espacio entre elementos. Es transparente.

🔲 border-radius

Sirve para redondear las esquinas. Con border-radius: 50% puedes hacer un círculo perfecto. ¡Muy útil para imágenes de perfil!

Sección 14

Flexbox — Diseño flexible

Flexbox es la herramienta más poderosa de CSS para organizar elementos en una fila o columna. Cuando la usas en un contenedor, los hijos se convierten en "flex items".

CSS — Flexbox básico
.contenedor {
  display: flex;                   /* ¡Activa Flexbox! */
  
  /* Dirección: fila (por defecto) o columna */
  flex-direction: row;            /* row | column */
  
  /* Alineación HORIZONTAL (en row) */
  justify-content: center;       /* flex-start | center | flex-end | space-between | space-around */
  
  /* Alineación VERTICAL (en row) */
  align-items: center;           /* flex-start | center | flex-end | stretch */
  
  /* Espacio entre elementos */
  gap: 20px;
  
  /* Permitir que los items se envuelvan */
  flex-wrap: wrap;
}

Casos de uso comunes

CSS — Recetas Flexbox
/* ① Centrar algo perfectamente */
.centrar-todo {
  display: flex;
  justify-content: center;
  align-items: center;
  height: 100vh; /* Ocupa toda la pantalla */
}

/* ② Navegación con logo a la izquierda y links a la derecha */
nav {
  display: flex;
  justify-content: space-between;
  align-items: center;
  padding: 16px 40px;
}

/* ③ Tarjetas que se ordenan solas */
.tarjetas {
  display: flex;
  flex-wrap: wrap;
  gap: 20px;
  justify-content: center;
}

Sección 15

CSS Grid — Diseño en cuadrícula

CSS Grid permite crear cuadrículas (como tablas) de forma flexible. Es ideal para layouts de páginas completas.

CSS — Grid básico
.contenedor {
  display: grid;

  /* 3 columnas iguales */
  grid-template-columns: 1fr 1fr 1fr;
  
  /* Equivalente más corto */
  grid-template-columns: repeat(3, 1fr);

  /* Columnas de tamaño fijo */
  grid-template-columns: 200px 1fr 200px;  /* sidebar-contenido-sidebar */

  /* Espacio entre celdas */
  gap: 20px;
}

/* Columnas automáticas que se adaptan */
.galeria-auto {
  display: grid;
  grid-template-columns: repeat(auto-fill, minmax(200px, 1fr));
  gap: 16px;
}
💡 ¿Cuándo usar Flexbox vs Grid? Flexbox: cuando organizas elementos en una sola dirección (fila O columna): navbars, botones juntos, tarjetas en fila. Grid: cuando necesitas una cuadrícula bidimensional (filas Y columnas): galerías de fotos, layouts de página completa.
CSS — Layout de página completa con Grid
body {
  display: grid;
  grid-template-areas:
    "header  header"
    "sidebar main"
    "footer  footer";
  grid-template-columns: 250px 1fr;
  grid-template-rows: auto 1fr auto;
  min-height: 100vh;
}

header { grid-area: header; background: #3498db; }
aside  { grid-area: sidebar; background: #ecf0f1; }
main   { grid-area: main; }
footer { grid-area: footer; background: #333; }

Sección 16

Diseño Responsivo

Un diseño responsivo se adapta automáticamente a cualquier tamaño de pantalla: celular, tablet o computadora.

Media Queries — La clave del responsive

CSS — Media Queries
/* Estilos base (mobile first — para celular primero) */
.contenedor {
  width: 100%;
  padding: 16px;
  font-size: 16px;
}

/* Para tablets (pantallas de 768px o más) */
@media (min-width: 768px) {
  .contenedor {
    padding: 32px;
    font-size: 18px;
  }
}

/* Para computadoras (pantallas de 1024px o más) */
@media (min-width: 1024px) {
  .contenedor {
    max-width: 1200px;
    margin: 0 auto;
  }
}

/* Cambiar a columna en celular */
.galeria {
  display: flex;
  gap: 20px;
  flex-direction: column;  /* columna en celular */
}

@media (min-width: 768px) {
  .galeria {
    flex-direction: row;     /* fila en tablet/PC */
  }
}
